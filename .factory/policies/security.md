# セキュリティポリシー

## 基本原則

本プロジェクトは業務利用を前提としており、セキュリティと監査可能性を最優先とします。

## 禁止事項

### 1. 機密情報の扱い
- API キー、パスワード、トークンをコードに直接埋め込まない
- 環境変数または安全な設定ファイルを使用する
- ログやコンソール出力に機密情報を含めない

### 2. 外部通信
- 明示的な承認なしに外部 API やサービスへの接続を行わない
- ネットワークリクエストは必ずログに記録する
- 証明書の検証を無効化しない

### 3. ファイルシステム操作
- システムディレクトリへの書き込みを行わない
- ユーザーデータの削除は必ず確認を求める
- 実行可能ファイルの動的生成を避ける

## 許可される操作

### 1. ローカルファイル操作
- プロジェクトディレクトリ内の読み書き
- ログおよび結果ファイルの出力
- 設定ファイルの読み込み

### 2. CLI実行
- DROID CLI の実行
- Git コマンドの実行
- テストフレームワークの実行

## コードレビュー時の確認事項

- [ ] 機密情報がハードコードされていないか
- [ ] 入力値の検証が適切に行われているか
- [ ] エラーメッセージに機密情報が含まれていないか
- [ ] ファイルパスがユーザー入力で直接構成されていないか
- [ ] 外部コマンド実行時にインジェクション対策が施されているか

## インシデント対応

セキュリティ上の問題を発見した場合：
1. 即座に問題を隔離する
2. 影響範囲を特定する
3. 修正内容をドキュメント化する
4. 再発防止策を講じる
