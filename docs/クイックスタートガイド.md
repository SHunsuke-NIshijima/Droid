# DROID Desktop GUI - クイックスタートガイド

## 🚀 すぐに始める

### 1. 起動
```cmd
run-droid.bat をダブルクリック
```

### 2. 設定
GUIが起動したら、以下を入力：

```
┌─────────────────────────────────────┐
│ プロンプト (Prompt) *                │
│ ここにDROIDへの指示を入力            │
│                                     │
│ 例: このコードをレビューして         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 作業ディレクトリ:  [.]      [参照]  │
│ モデル:            [claude-sonnet▼] │
│ 自動化レベル:      [medium      ▼]  │
└─────────────────────────────────────┘
```

### 3. 実行
```
右下の [実行] ボタンをクリック
```

## 📋 主な機能

| 機能 | 説明 | 使い方 |
|------|------|--------|
| **プロンプト入力** | DROIDへの指示 | 複数行入力可能 |
| **作業ディレクトリ** | 実行場所 | [参照]ボタンで選択 |
| **モデル選択** | AIモデル | ドロップダウンで選択 |
| **自動化レベル** | 承認レベル | low/medium/high |
| **参照ファイル** | 参照するファイル/フォルダ | [ファイル追加]/[フォルダ追加] |
| **保存** | 設定を保存 | prompt.jsonに保存 |
| **実行** | DROIDを実行 | 保存＋実行 |

## 💡 よく使うパターン

### パターン1: コードレビュー
```
プロンプト: このコードをレビューして改善点を教えて
参照ファイル: src/main.py を追加
→ [実行]
```

### パターン2: バグ修正
```
プロンプト: エラーを修正して
参照ファイル: 修正対象ファイルを追加
自動化レベル: high
→ [実行]
```

### パターン3: コード作成
```
プロンプト: Pythonで計算機アプリを作成して
作業ディレクトリ: 作成先を指定
→ [実行]
```

## 🎯 モデルの選び方

| モデル | 速度 | 品質 | コスト | 推奨用途 |
|--------|------|------|--------|----------|
| **claude-opus-4-5** | 遅い | 最高 | 高 | 複雑なタスク |
| **claude-sonnet-4** | 普通 | 高 | 中 | **通常用途（推奨）** |
| **claude-haiku-4** | 速い | 普通 | 低 | 簡単なタスク |

## ⚙️ 自動化レベルの選び方

| レベル | 動作 | 推奨シーン |
|--------|------|------------|
| **low** | 全て手動確認 | 重要な変更 |
| **medium** | 一部自動承認 | **通常用途（推奨）** |
| **high** | 全て自動承認 | テスト環境 |

## 🔍 トラブルシューティング

### Q: GUIが起動しない
```
A: コマンドプロンプトから実行してエラーを確認
   cd /d "C:\path\to\Droid"
   python gui_droid.py
```

### Q: 「Pythonがインストールされていません」
```
A: Python 3.8以上をインストール
   https://www.python.org/downloads/
```

### Q: 「droidコマンドが見つからない」
```
A: DROIDがインストールされているか確認
   コマンドプロンプトで: droid --version
```

### Q: 実行が完了しない
```
A: logsフォルダ内のログファイルを確認
   大きなプロジェクトは時間がかかります
```

## 📁 ファイルの場所

```
Droid/
├── run-droid.bat     ← これを実行
├── gui_droid.py      ← GUIアプリ本体
├── prompt.json       ← 設定ファイル（自動生成）
└── logs/             ← ログファイル（自動生成）
```
